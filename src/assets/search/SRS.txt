Software Requirement Specification (SRS)

Project Name: AI-Powered Nelna Product Finder (Location-Based Smart Search System)
Prepared By: Chinthaka Jayaweera
Date: 29-Jan-2026

1. Introduction

1.1 Purpose
The purpose of this system is to enable customers of Nelna, a Sri Lankan poultry and frozen food brand, to quickly and easily find the nearest stores selling their desired products using AI-powered natural language search combined with geolocation. This system aims to enhance customer experience, reduce product search time, and provide valuable business insights.

1.2 Scope
The system will:

* Allow users to search for Nelna products using natural language queries.
* Provide location-based store recommendations.
* Rank results using AI semantic relevance and distance.
* Integrate Google Maps for store visualization.
* Offer product recommendations based on AI analysis.
* Support mobile-first web experience.

Out of Scope:

* Real-time inventory updates from all stores (initial MVP will use preloaded stock data).
* Advanced AI chat assistant (optional for later phases).

1.3 Definitions, Acronyms, Abbreviations

* AI: Artificial Intelligence
* MVP: Minimum Viable Product
* API: Application Programming Interface
* DB: Database
* GeoNear: MongoDB geospatial query
* Embedding: AI-generated vector representation of text for semantic search

2. Overall Description

2.1 Product Perspective
The system is a web-based application integrating frontend, backend, database, AI embeddings, and Google Maps services. It serves as a standalone product discovery platform for Nelna customers.

System Components:

* Frontend (React + TypeScript + Tailwind CSS + ShadCN UI)
* Backend (Node.js + Express API)
* Database (MongoDB Atlas with Vector + Geo indexes)
* AI Service (OpenAI Embeddings API)
* Maps Service (Google Maps API)

2.2 Product Functions

1. Natural language search for Nelna products.
2. AI-based semantic search for product matching.
3. Geolocation-based store search and ranking.
4. Display of store details and product availability.
5. Google Maps integration with pins and directions.
6. Product recommendations (related items).
7. Optional: AI chat interface for product queries.

2.3 User Characteristics
Primary Users:

* Customers of Nelna looking for nearby products.
* Users with basic web browsing skills.

Secondary Users:

* Nelna marketing & business analysts for store/product insights.

2.4 Constraints

* Must work on mobile and desktop.
* Backend must handle concurrent queries efficiently.
* AI search latency must be under 2 seconds.
* Secure storage of any user input data.
* MVP must avoid real-time stock integration.

2.5 Assumptions & Dependencies

* User devices have browser geolocation enabled.
* OpenAI API and Google Maps API keys are available.
* MongoDB Atlas is accessible and properly indexed.

3. Specific Requirements

3.1 Functional Requirements

| ID   | Requirement             | Description                                                    |
| ---- | ----------------------- | -------------------------------------------------------------- |
| FR-1 | Search Query            | Users enter natural language queries for products.             |
| FR-2 | AI Semantic Search      | Convert query to embedding and find relevant products.         |
| FR-3 | Store Matching          | Find stores in the user's location offering selected products. |
| FR-4 | Result Ranking          | Rank results by AI relevance score + distance score.           |
| FR-5 | Map Integration         | Display store locations on Google Maps.                        |
| FR-6 | Product Recommendations | Suggest related products to users.                             |
| FR-7 | Optional Chat           | AI chat interface for product queries.                         |

3.2 Non-Functional Requirements

| Category        | Requirement                                                      |
| --------------- | ---------------------------------------------------------------- |
| Performance     | Query response < 2 seconds.                                      |
| Reliability     | 99% uptime for backend APIs.                                     |
| Scalability     | Must support up to 100,000 products and 10,000 stores in future. |
| Security        | Store API keys securely, protect user data.                      |
| Usability       | Mobile-first responsive design.                                  |
| Maintainability | Clean code, modular architecture.                                |

3.3 Database Requirements
Products Collection:

* productCode: String
* name: String
* category: String
* description: String
* embedding: Array (vector for semantic search)

Stores Collection:

* storeName: String
* city: String
* location: GeoJSON (lat/lng)
* availableProducts: Array of productCodes
* embedding: Array (vector for semantic search)
* contact: String

Indexes:

* Vector index on embeddings
* 2dsphere index on store location

3.4 API Requirements
Endpoint: /api/ai-search
Method: POST
Request:
{
"query": "Chicken sausages near Colombo",
"userLocation": { "lat": 6.9271, "lng": 79.8612 }
}

Response:
{
"products": [
{ "productCode": "07NE05", "name": "Nelna EASY 5Kg", "score": 0.98 }
],
"stores": [
{ "storeName": "Cargills Kandy", "distance": 2.1, "availableProducts": ["07NE05"] }
]
}

3.5 AI Search Logic

1. Convert user query → embedding using OpenAI API.
2. Retrieve top-N relevant products using vector search.
3. Retrieve stores in the user’s city or nearby locations.
4. Calculate final ranking score:
   FinalScore = (AI_Relevance * 0.6) + (DistanceScore * 0.4)
5. Return sorted results with store details and map coordinates.

3.6 UI Requirements

* Search bar (natural language input)
* Use current location button
* Result cards showing store name, product, distance, contact
* Map view toggle (Google Maps)
* Mobile-first responsive design

3.7 Scalability & Security

* Secure API keys and credentials using environment variables.
* Use vector indexes for fast AI searches.
* Use 2dsphere indexes for geospatial queries.
* Backend supports horizontal scaling (stateless APIs).
* Validate all user inputs.

4. Deliverables

* Fully functional web application MVP
* Backend API endpoints
* Database schema with indexes
* AI embeddings and search logic
* React frontend with search UI & Google Maps
* Documentation (SRS + API + Architecture)

```components/search/SearchBar.tsx
import React, { useState } from 'react'
import { Search, MapPin, Mic } from 'lucide-react'
import { Button } from '../ui/Button'
interface SearchBarProps {
  onSearch: (query: string) => void
  onLocationClick: () => void
  isLocating: boolean
  locationAddress?: string
}
export function SearchBar({
  onSearch,
  onLocationClick,
  isLocating,
  locationAddress,
}: SearchBarProps) {
  const [query, setQuery] = useState('')
  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault()
    if (query.trim()) {
      onSearch(query)
    }
  }
  return (
    <div className="w-full max-w-3xl mx-auto">
      <form onSubmit={handleSubmit} className="relative group">
        <div className="absolute inset-y-0 left-0 pl-4 flex items-center pointer-events-none">
          <Search className="h-5 w-5 text-gray-400 group-focus-within:text-[#C41E3A] transition-colors" />

          type="text"

          placeholder="What are you looking for today?"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
        />

        <div className="absolute inset-y-0 right-2 flex items-center space-x-1">
          <button
            type="button"
```

```components/ui/Button.tsx
import React from 'react'
import { Loader2 } from 'lucide-react'
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary' | 'outline' | 'ghost'
  size?: 'sm' | 'md' | 'lg'
  isLoading?: boolean
  leftIcon?: React.ReactNode
  rightIcon?: React.ReactNode
}
export function Button({
  children,
  className = '',
  variant = 'primary',
  size = 'md',
  isLoading = false,
  leftIcon,
  rightIcon,
  disabled,
  ...props
}: ButtonProps) {
  const baseStyles =
    'inline-flex items-center justify-center rounded-lg font-medium transition-all focus:outline-none focus:ring-2 focus:ring-offset-2 disabled:opacity-50 disabled:cursor-not-allowed'
  const variants = {
    primary: 'bg-[#C41E3A] text-white hover:bg-[#A01830] focus:ring-[#C41E3A]',
    secondary:
      'bg-[#FF6B35] text-white hover:bg-[#E55A25] focus:ring-[#FF6B35]',
    outline:
      'border-2 border-[#C41E3A] text-[#C41E3A] hover:bg-[#FFF9F5] focus:ring-[#C41E3A]',
    ghost:
      'text-gray-600 hover:bg-gray-100 hover:text-gray-900 focus:ring-gray-500',
  }
  const sizes = {
    sm: 'h-9 px-3 text-sm',
    md: 'h-11 px-5 text-base',
    lg: 'h-14 px-8 text-lg',
  }
  return (
    <button
      className={`${baseStyles} ${variants[variant]} ${sizes[size]} ${className}`}
      disabled={disabled || isLoading}
      {...props}
    >
      {isLoading && <Loader2 className="mr-2 h-4 w-4 animate-spin" />}
      {!isLoading && leftIcon && <span className="mr-2">{leftIcon}</span>}
      {children}
      {!isLoading && rightIcon && <span className="ml-2">{rightIcon}</span>}
    </button>
  )
}
```

```hooks/useGeolocation.ts
import { useState, useEffect } from 'react'
import { Location } from '../types'

interface GeolocationState {
  location: Location | null
  error: string | null
  loading: boolean
}

export function useGeolocation() {
  const [state, setState] = useState<GeolocationState>({
    location: null,
    error: null,
    loading: false,
  })

  const getLocation = () => {
    if (!navigator.geolocation) {
      setState((prev) => ({
        ...prev,
        error: 'Geolocation is not supported by your browser',
      }))
      return
    }

    setState((prev) => ({ ...prev, loading: true, error: null }))

    navigator.geolocation.getCurrentPosition(
      (position) => {
        setState({
          location: {
            lat: position.coords.latitude,
            lng: position.coords.longitude,
          },
          error: null,
          loading: false,
        })
      },
      (error) => {
        setState({
          location: null,
          error: error.message,
          loading: false,
        })
      },
    )
  }

  return { ...state, getLocation }
}
```

```hooks/useProductSearch.ts
export function useProductSearch() {
  const [loading, setLoading] = useState(false)
  const [results, setResults] = useState<SearchResult | null>(null)
  const [error, setError] = useState<string | null>(null)

  const search = async (
    query: string,
    location: { lat: number; lng: number } | null,
  ) => {
    setLoading(true)
    setError(null)

    try {
      // Simulate API delay
      await new Promise((resolve) => setTimeout(resolve, 1500))

      // Simple mock logic: filter if query matches, otherwise return all
      const relevantProducts = query
        ? MOCK_PRODUCTS.filter(
            (p) =>
              p.name.toLowerCase().includes(query.toLowerCase()) ||
              p.category.toLowerCase().includes(query.toLowerCase()),
          )
        : MOCK_PRODUCTS

      // Sort stores by distance (mock)
      const sortedStores = [...MOCK_STORES].sort(
        (a, b) => a.distance - b.distance,
      )

      setResults({
        products: relevantProducts,
        stores: sortedStores,
        relatedProducts: MOCK_PRODUCTS.filter(
          (p) => !relevantProducts.includes(p),
        ).slice(0, 2),
      })
    } catch (err) {
      setError('Failed to fetch results. Please try again.')
    } finally {
      setLoading(false)
    }
  }

  return { search, results, loading, error }
}
```

```types/index.ts
  name: string

  distance: number // in km
  availableProducts: string[] // product IDs
  contact: string
  isOpen: boolean
}

export interface SearchResult {
  products: Product[]
  stores: Store[]
  relatedProducts: Product[]
}
```
